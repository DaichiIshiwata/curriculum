■ネットワーク層

IP　InternetProtocol ネットワーク層で動作するプロトコル ネットワーク層のルール

IPヘッダ　IPパケットの冒頭部にある、IPバージョン・送信元アドレス・宛先アドレスがある情報部分。IPのときに追加される ネットワーク層で追加される最初の部分

ICMP　Internet Control Message Protocol ネットワーク通信の問題を診断するために必要なプロトコル

ICMPメッセージ 問い合わせ(Query)とエラー通知(Error)のこと

IPアドレス　インターネット上での住所

	オクテット　8ビットを表している。8ビット以外を表すことがあるバイトの代わりに確実に8ビットを表しているもの 2進数にしたときに分けてるやつ
	
	ネットワーク部　そのIPアドレスが属しているネットワークを識別する部分 どこのクラスですよーって感じ

	ホスト部　ネットワーク内のコンピュータを認識する部分 どこのクラスのだれーまでわかる部分
	
	クラス　どこまでがネットワークアドレスか、ホストアドレスかがわかるようにするもの　区切りの部分

	プライベートIPアドレス LANに接続された組織内のネットワーク　電話でおける内線番号のようなもの

	グローバルIPアドレス　インターネットに接続する際に割り当てられるIPアドレス　世界で一つしかないアドレス

ネットワークアドレス　あるネットワークに割り当てられたIPアドレスの中で、そのネットワーク自身を示すアドレス 所属を表すもの

ブロードキャストアドレス　ネットワーク内のすべての機器にデータを一斉配信するためのアドレス　みんなに送るよっていうやつ

ループバックアドレス　自分自身に対してデータを送信すること。自分自身を表すIPアドレス

サブネット ネットワーク内のネットワーク ネットワークを分けた内のネットワークのこと ○○市の○○丁目とかそういう

サブネットマスク　ネットワークの範囲を定義するために使う　

	プレフィックス表記 CIDR表記と同じ　「〇〇.○○.○○.○○/24」のような、IPアドレスとサブネットマスクを同時に表記する書き方のこと

デフォルトゲートウェイ 標準もしくは基本となる出入口。閉じられたネットワークから外部のネットワークへ接続するための、基本となる出入口が設定された、ルーターやサーバーのこと

IPv6　「Internet Protocol Version6」の略でインターネットプロトコルの規格
	
	サブネットプレフィックス　IPv6で、アドレスの種類やネットワークの識別子として用いられる前半部分が「プレフィックス」という。

	インターフェイスID　 IPv6アドレスでの後半部分。ネットワーク内の識別子などを表す



ルータ　インターネットなどのネットワーク通信を行うための中継機器。どういう経路でデータを排sんするとか考慮しながら、情報の制御や優先をスムーズに選択してくれるもの。

	コンソール接続　コンピュータからシリアルケーブルなどを介して機器に接続し、文字による対話的な操作のこと

	VTY接続　vritual teletype　ルータなど通信機器にネットワークを通じて別のコンピュータから接続して、操作や管理を行う仮想的な接続口

	AUX接続 モデムを通じて公衆回線を利用して機器にアクセスする方法。通常のWAN側の回避が利用できない場合や機器同士を直接接続して、他の機器からログインできるようにする際に使う

	ルーティングテーブル　ルーター内で経路情報を示す経路図 このパケットを正しく届けるに次はどのルーターにデータを引き渡すと効率的かが書いてあるテーブル

		ネクストホップ　受け取った通信を次にどのルータに転送するかを示したもの

	ARPテーブル　ARP(Address Resolution Protocol)によって割り出したIPアドレスとMACアドレスの対応表
	
		ARP　Address Resolution Protocol　アドレス解決プロトコル インターネットや社内LANなどに利用されてるパケットを送信する時、宛先となる宛先のMACアドレスを割り出せるプロトコル

	コアルータ　通信キャリアやISPネットワークの通信を中継するために使われるルータ

	センタールータ　企業の拠点間のネットワークにおいて拠点内のネットワークを接続するために利用する高い性能を持ったルータ　データセンタや本社拠点にはネットワークトラフィックが集中するため、こちらが使われる

	エッジルータ　企業の拠点間のネットワークにおいて拠点内のネットワークを接続するために利用する高い性能を持ったルータ　支社や営業所等の通信が少ない拠点についてはこちらが使われる
	
	ブロードバンドルータ　主にインターネット回線に接続するために利用されるもの インターネット接続を複数のデバイスで共有するためのインターネット機器



ルーティング　ネットワーク上でデータを転送する際、その経路を導き出すこと　ネットワークで行われる道案内

	直接接続　あるコンピューターがネットワークではなく、ケーブルによって別のコンピューターに直接リンクされている状況のこと　中間のルーターやアクセスポイントを必要とせずデバイス間の通信を可能にする方法

	スタティックルーティング　(静的ルーティング)　あらかじめネットワーク管理者が設定したルーティングテーブルに則ってルーティングを行うプロセス

	ダイナミックルーティング　(動的ルーティング)　ルーター同士が常にルート情報を交換しあい、ルーティングテーブルの最適化を動的に行うプロセス

		ディスタンスベクタ型　ディスタンス(距離)とベクタ(方向)により最適なルートを決定する手順 ルータ間では定期的なルーティングテーブル(経路表)全体を交換し合うことでリモートネットワークの経路情報を受信する。単純な動作であり収束が遅い。「RIP」「IGRP」がある

		リンクステート型　ルーティングプロトコルの特徴・分類の一つで、どのルータとどのルータが隣接しているのかという接続情報(リンクステート)を交換し合い、情報の集合に基づいて経路を選択する。「OSPF」や「IS-IS」などがある

		ハイブリッド型　ディスタンスベクタ型の利点とリンクステート型の利点を組み合わせた(Hybrid)方式。基本としてはディスタンスベクタ型なので拡張ディスタンスベクタ型とも呼ばれる。「EIGRP」がある

■トランクポート層　データの送受信を行い端末どうしてデータが正しく手順を踏んで送り届けられたか、という管理と信頼性の確保のための方式が規定されている。ネットワーク層を通して送られてきたデータの整序や誤りの訂正、および再送の要求などの動作が行われる

コネクション型通信　connection-oriented communication(ストリーム型通信)　データ通信において、通信を開始する前に相手との間で仮想的な専用通信路(コネクション)を確立し、それを通じてデータの送受信を行う方式。　TCPがこれを行う

	ACK　ACKnowledgement とは二者間の通信で、あいい手形に何らかの肯定的な応答を返す際に送られる信号やデータ、パケットなどのこと。「接続要求を受理した」「データが正しく受信出来た」といった内容を伝達するために使われる

コネクションレス型通信　connectionless communication とはデータ通信において、通信相手の状況を確認せずデータを一般的に送り付ける通信方式。またお互いにそのようにデータを送りあう方式。送信したデータが相手に届くかどうかや、送信したデータの到着順は保証されない。IPネットワークではUDP(User Datagram Protocol)が通信を行う

	TCP　IPと同様にインターネットにおいて標準的に利用されているプロトコル。IPの上位プロトコルでトランクポート層で動作する。ネットワーク層のプロトコルとセッション層以上のプロトコルの橋渡しをする。通信相手の状況を確認してデータが届いたか確認しながら送りあう方式。信頼性は高いが転送速度が遅い。　

	スリーウェイハンドシェイク　TCPはコネクション型プロトコルであることから、データ転送を行う前にコネクションの確立を行う。コネクション確立のことをスリーウェイハンドシェイクという。3回のやりとりで確立される

	Telnet　インターネットやLANなどのネットワークにおいて、ネットワークに繋がれたコンピュータを遠隔操作するための仕組み

	HTTP　HyperText Transfer Protocol　通信する時に使う通信プロトコルの一つ。ホームページのファイルとかを受け流しする時に使う通信プロトコル

	FTP　File Transfer Protocol ファイルを転送するための通信規格。クライアントとサーバ間で、ファイルのアップロードやダウンロードを行うときに使われるプロトコル

	SMTP　Simple Mail Transfer Protocol メールサーバが送信元と宛先の間でメールの送受信や中継を行うために使用されている。メール通信という仕組みを裏から技術的に支えているもので、メールの送受信を可能にするプロトコル

TCPヘッダ　TCPセグメントの先頭部分にある、制御方法などを一定の形式で記述した領域。この後ろに送りたいデータ本体(ペイロード)を加えて、1つのTCPセグメント　内容が固定されている部分が先頭から20バイト。オプションが4バイト単位の可変長となっている

UDP　User Datagram Protocol インターネット上で動画の再生や、DNSのルックアップなど、特に時間を重視した通信に使用される通信プロトコル。データ転送前に正式な接続の確立をしないことで、通信を高速化している。

	RTP　Real-time Transport Protocol TCP/IPネットワーク上で音声や動画のように連続するデータの流れをリアルタイムに伝送するための通信プロトコルの一つ。分割された連続歴なデータの送受信についてのみ定めたシンプルなプロトコル。データの到着順の保証や欠落時の再送制御など省略する代わりに拘束に伝送することができる。

	SNMP　Simpel Network Management Protocol　ルータ、スイッチ、サーバなどTCP/IPネットワークに接続された通信機器に対し、ネットワーク経由で監視、制御するためにアプリケーション層プロトコルです。SNMPを利用することにより、導入したネットワーク機器を監視することができるので、ネットワークに障害が発生した場合にどの危機に障害が発生したのかを、すぐに突き止められる。

	DHCP　Dynamic Host Configuration Protocol　コンピュータがネットワークに接続して通信する際に必要な設定情報を自動的に割り当てる機能
管理者側で適切な設定を自動的に適用でき、技術に詳しくない利用者でも簡単に接続できる。管理者は多くの機器の設定を容易に一元管理することができ、不適切な設定に起因するトラブルを減らすことができる。

	TFTP　Trivial File Transfer Protocol　IPネットワーク上でファイル転送を行うためにプロトコルの一つ。ファイルの自動配布などのために用いられるもので、認証などのない簡易な設定なもの。標準的にはクライアントからサーバの「UDP69番ポート」にアクセスして通信を開始し、お互いに空いているポートを通知しあってポート間でデータ転送。

	RIP　Routing Information Protocol IPネットワークで用いられるルーティングプロトコルの一つ。ルータなどの通信機器の間で経路情報を交換し、ある地点から別の地点まで最短経路を割り出すのに使われる。自立システム(AS:Autonomous System)の内部で経路選択を行うIGP(Interior Gateway Protocol)として使われる。ディスタンスベクタ型を使っている。

UDPヘッダ　UDPデータグラムの先頭部分にある。制御情報などを一定の形式で記述した領域。この後ろに送りたいデータ本体を加えて、1つのUDPデータグラムとなる。
